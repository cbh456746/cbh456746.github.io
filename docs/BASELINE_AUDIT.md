# 기준선 감사 보고서

> 상태: Phase 0 기준 문서  
> 대상: `cbh456746.github.io-main` 압축본  
> 원칙: 이 문서는 현 상태를 기록한다. 이 문서만으로 배포 설정, 콘텐츠, URL, 테마를 변경하지 않는다.

## 1. 저장소의 현재 성격

이 저장소는 Hux Blog 기반의 Jekyll 정적 사이트다. 현재 역할은 개인 블로그에 가깝지만, 앞으로는 다음을 함께 담는 개인 플랫폼으로 확장한다.

- 한 장짜리 이력 및 전문 소개
- 프로젝트 아카이브
- 연구·학습 노트
- `Play` 중심의 자유로운 기록

현재 배포와 렌더링의 핵심 흐름은 다음과 같다.

```text
_config.yml
  -> _layouts/default.html
     -> _includes/head.html, nav.html, search.html, footer.html
  -> _layouts/page.html (홈·About)
  -> _layouts/post.html (게시물·목차·MathJax·댓글)
  -> _posts/ (홈·아카이브·검색·RSS의 원천)
```

## 2. 확인된 사실

| 항목 | 현재 상태 | 영향 |
|---|---|---|
| 콘텐츠 | Markdown 게시물 13개, 프로젝트/연구/일상이 모두 `_posts`에 혼재 | 목적별 탐색과 확장이 어려움 |
| 게시물 파일명 | 10개가 `25-03-...` 형식 | Jekyll 표준 게시물 인식 실패 가능성 |
| 프런트매터 | 3개 게시물에 없음 | 날짜·레이아웃·태그·SEO의 일관성 상실 |
| 태그 | 주로 `Data Science`, `Meta` | 콘텐츠 성격을 구분하지 못함 |
| 홈 | 페이지 레이아웃에 최근 게시물 목록을 출력 | 개인의 자유로운 표현 공간이라는 새 목적과 다름 |
| 소개 | About 한 페이지에 한/영 본문을 함께 둠 | 이력 정보 관리와 다국어 운영이 불명확 |
| 빌드 | Jekyll + `jekyll-paginate`, Grunt + Less | Node/Grunt와 Ruby/Jekyll 책임이 혼재 |
| 배포 자동화 | GitHub Actions 없음 | 변경 검증과 재현 가능한 배포가 없음 |
| 정적 자산 | CSS/JS 원본과 minified 파일이 함께 있음 | 어떤 파일이 편집 원본인지 명확하지 않음 |

## 3. 최우선 위험 요소

### 배포 및 정체성

- `CNAME`은 원 테마 도메인인 `huangxuan.me`를 가리킨다. 실제 소유·사용 여부를 확인하기 전에는 절대 수정하거나 배포하지 않는다.
- `_config.yml`에 원 테마의 설명, 키워드, Analytics 도메인, PWA 이름·설명이 남아 있다.
- 푸터에는 Hux Blog 링크와 원 테마 GitHub 스타 iframe이 있다.
- `_doc/`는 이 프로젝트의 문서가 아니라 원 테마의 문서다. 새 운영 문서는 루트 `docs/`에 둔다.

### 콘텐츠 공개

- 프런트매터가 없는 파일: 로컬 GitHub Desktop 안내, 네이버 금융 리포트 크롤러, 인구 피라미드 시각화.
- 파일명·날짜가 서로 다른 글이 있다. 예: `2025-03-15-hello_trans.md`의 본문 날짜는 2015년이다.
- Bugs 가사 크롤러와 가설검정 글은 파일명에서 예상되는 연도와 프런트매터 날짜가 다르다.
- PCA 제목에는 닫는 괄호 누락이 있다.

### 사용자 경험 및 유지보수

- 홈에 최근 글 목록과 개인 소개가 결합되어 있다. 새 요구사항은 홈을 자유로운 Soul 페이지로, 이력을 별도 페이지로 분리한다.
- 검색 폼에는 중복 닫는 태그와 접근성 레이블 누락이 있다.
- 모바일 메뉴와 이미지 대체 텍스트에 접근성 개선이 필요하다.
- 서비스 워커가 루트 경로를 하드코딩하고, 캐시 자산 일부가 현재 설정과 맞지 않으며 메시지 처리 코드에 변수 오류가 있다.

## 4. 현재 기능의 유지 여부

| 기능 | 기본 결정 | 후속 작업 |
|---|---|---|
| Markdown | 유지 | 작성 템플릿과 검사 추가 |
| LaTeX/MathJax | 유지·현대화 | 필요한 문서에서만 로드 |
| 코드 하이라이트 | 유지 | 가독성 있는 테마로 정리 |
| 검색 | 유지·개선 | 태그 인덱스와 연동 |
| 태그 아카이브 | 유지·재설계 | 좌측 인덱스와 제목 목록 제공 |
| RSS | 유지 | 실제 사이트 메타데이터로 교정 |
| PWA | 보류 | 캐시 마이그레이션 설계 후 유지/제거 결정 |
| Disqus | 보류 | 실제 사용 의사·개인정보 영향 확인 후 결정 |
| AdSense | 보류 | 수익화 필요성과 사용자 경험을 검토 후 결정 |
| 구형 Analytics | 교체 또는 제거 | 개인정보 정책과 측정 목적을 먼저 확정 |

## 5. 금지 사항

- URL 매핑표가 확정되기 전 게시물 파일을 이름만 보고 이동·삭제하지 않는다.
- 도메인 소유 및 DNS 상태를 확인하기 전 `CNAME`을 임의로 교체하지 않는다.
- 서비스 워커를 수정할 때는 기존 사용자의 캐시를 정리하는 마이그레이션을 포함한다.
- 원 테마의 Apache-2.0 저작권·NOTICE를 제거하지 않는다.
- 새로운 콘텐츠는 `Blog` 태그를 사용하지 않는다. 자유 기록은 `Play`, 게임 관련 기록은 `Game`을 사용한다.
